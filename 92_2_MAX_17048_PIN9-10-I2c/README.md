# MAX17048 на PIN 9/10 (Wire1 / SoftI2C)

Экспериментальные версии драйвера MAX17048 для пинов **9 (PF2, I2C1_SDA)** и **10 (PF1, I2C1_SCL)**.

Основная рабочая версия (Pin 12/13, Wire) находится в корне репозитория. Эта папка — для случаев, когда нужно освободить Pin 12/13 под другие задачи.

---

## Анализ вариантов (Pin 9/10)

### Вариант 1: Wire1 (аппаратный I2C1)

На AMB82-mini два аппаратных I2C:
- **I2C0** → Arduino `Wire` → Pin 12 (SDA) / 13 (SCL)
- **I2C1** → Arduino `Wire1` → Pin 9 (SDA) / 10 (SCL)

**Проблема:** в SDK Ameba Pro2 v4.0.7–4.0.9 `Wire1` имеет подтверждённые баги:
- Конфликт с DMIC подсистемой → `[MISC Err] Pin 2[26] is conflicted`
- `setClock(50000)` валит SDK в **Bus Fault / HardFault**
- Сканер показывает "фантомные" устройства (0x36…0x7E), но чтение регистров нестабильно

**Возможное решение:** обновить Ameba boards package в Arduino IDE до более свежей версии (если вышла после 4.0.9). Ребята из Realtek могли починить.

**Файл:** `92_2_MAX_17048_Wire1.ino`

---

### Вариант 2: SoftI2C (bit-bang)

Если `Wire1` всё ещё сломан — программный I2C на тех же пинах 9/10.

**Важно:** пины 9/10 **могут работать как GPIO** (`digitalWrite(9, HIGH)` даёт 3.3V — проверено). `Wire1` не блокирует их, если `Wire1.begin()` не вызывать.

**Улучшения по сравнению с первой попыткой:**
- Open-drain эмуляция через `OUTPUT` + `digitalWrite(HIGH/LOW)` (не переключать `pinMode` каждый бит)
- Внешние подтяжки уже есть на модуле (~9k между SDA-SCL = 2×4.7k)
- Медленная частота (~50 кГц) для стабильности
- Чип подтверждённо рабочий (читается через Wire 12/13)

**Файл:** `92_2_MAX_17048_SoftI2C.ino`

---

### Вариант 3: Перенести SPI на другие пины (альтернатива)

Вместо борьбы с I2C1 на 9/10 — оставить MAX17048 на рабочем `Wire` (12/13), а ADS1256 (SPI) перенести на **SPI0** (Pin 12-15: PE4/PE3/PE2/PE1).

**Но:** Pin 12/13 — shared между SPI и I2C. Если ADS1256 занимает SPI на 12-15, I2C на тех же пинах работать не будет. В текущем проекте ADS1256 на SPI1 (Pin 0-3), так что этот вариант не даёт преимуществ.

**Вывод:** проще всего либо починить Wire1 (обновить SDK), либо использовать SoftI2C.

---

## Рекомендуемый порядок действий

1. Попробовать `92_2_MAX_17048_Wire1.ino` — если сканер покажет только `0x36` и VERSION читается → **Wire1 работает**
2. Если Wire1 глючит → использовать `92_2_MAX_17048_SoftI2C.ino`
3. SoftI2C проверенно работает, если чип живой и подтяжки есть

## Чек-лист железа (для Pin 9/10)

| Проверка | Ожидаемо |
|----------|----------|
| SDA (Pin 9) ↔ GND при включённой Ameba | ~3.3V |
| SCL (Pin 10) ↔ GND при включённой Ameba | ~3.3V |
| SDA ↔ SCL при выключенном питании | ~9kΩ (подтяжки на модуле) |
| BAT+ / BAT- подключены к аккумулятору | Да |
| VCC модуля = 3.3V | Да |
